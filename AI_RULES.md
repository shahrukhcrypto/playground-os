# Playground OS - AI Development Rules

1. Read the entire codebase before making changes.
2. Read all documentation before starting work.
3. Never duplicate code.
4. Reuse existing components.
5. Keep the architecture modular.
6. Verify build before finishing.
7. Fix errors automatically.
8. Commit after every completed milestone.
9. Push to GitHub immediately after commit.
10. Stop only if a human decision is required or an unrecoverable blocker is encountered.
11. Prefer the most cost-effective solution.
12. Think long-term. Never implement quick hacks.
